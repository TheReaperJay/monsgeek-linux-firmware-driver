- M5W wired USB identity is `0x3151:0x4015`
- M5W dongle PID is `0x4011`
- `connect()` takes `&DeviceDefinition`
- `GET_USB_VERSION` carries a 32-bit device ID
- `udev` is the real hot-plug path in this environment
- `HID_QUIRK_IGNORE` is the chosen kernel-driver workaround on this host setup
- short-lived sessions must return `IF0` to the kernel
