    Define test constants at the top:
    ```rust
    const M5W_VID: u16 = 0x3141;
    const M5W_PID: u16 = 0x4005;
    ```

    **Test 1: test_enumerate_m5w** (HID-01)
    - Load DeviceRegistry from `../../crates/monsgeek-protocol/devices/`
    - Call `monsgeek_transport::discovery::enumerate_devices(&registry)`
    - Assert at least one DeviceInfo in the result
    - Assert the first result has vid == 0x3141 and pid == 0x4005

    **Test 2: test_get_usb_version** (HID-02)
    - Call `monsgeek_transport::connect(M5W_VID, M5W_PID)`
    - Send GET_USB_VERSION (0x8F) query with ChecksumType::Bit7 and empty data
    - Assert response[0] == 0x8F (echo byte)
    - Parse device ID from response bytes (reference: bytes 1-2 are device ID as little-endian u16, or bytes at known offset -- read from reference implementation to determine exact offset)
    - Log the full response for debugging
    - Call handle.shutdown()

    **Test 3: test_throttle_rapid_commands** (HID-03)
    - Connect to M5W
    - Send 5 GET_USB_VERSION queries in quick succession (no sleep between calls)
    - Measure total elapsed time
    - Assert total time >= 500ms (5 commands * 100ms minimum delay each)
    - All 5 should succeed (transport thread throttles them)
    - This proves the firmware doesn't crash from rapid commands

    **Test 4: test_echo_matching** (HID-04)
    - Connect to M5W
    - Send GET_USB_VERSION (0x8F) and verify response[0] == 0x8F
    - Send GET_DEBOUNCE (use protocol family detection to get correct command byte) and verify response[0] matches the sent command byte
    - This proves echo matching works across different command types

    **Test 5: test_bounds_validation** (HID-05)
    - Load DeviceRegistry, get M5W definition
    - Call `validate_write_request(&m5w, 0, 0)` -- should Ok (valid)
    - Call `validate_write_request(&m5w, 107, 3)` -- should Ok (max valid for 108 keys, 4 layers)
    - Call `validate_write_request(&m5w, 108, 0)` -- should Err (key_index == key_count)
    - Call `validate_write_request(&m5w, 0, 4)` -- should Err (layer == max_layers)
    - Note: This test doesn't require hardware, but is here to validate against the real M5W device definition from JSON

    **Test 6: test_udev_rules_file** (HID-06)
    - Assert that `deploy/99-monsgeek.rules` exists (path relative to crate root)
    - Read file contents, assert it contains `ATTRS{idVendor}=="3141"`
    - Assert it contains `TAG+="uaccess"` (non-root access)
    - Assert it contains `bInterfaceNumber}=="02"` (IF2 unbind)
    - Note: This test doesn't require hardware but validates the udev file is correct

    **Test 7: test_hot_plug_detection** (HID-01, hot-plug aspect)
    - Call `monsgeek_transport::connect(M5W_VID, M5W_PID)`
    - Log: "Hot-plug test: please unplug and replug the M5W keyboard within 30 seconds"
    - Wait on event_rx with 30-second timeout
    - If DeviceLeft received, log "Device left detected"
    - If DeviceArrived received after DeviceLeft, log "Device arrived detected"
    - Note: This test will be skipped if hot-plug is not supported on the platform (log and pass)
