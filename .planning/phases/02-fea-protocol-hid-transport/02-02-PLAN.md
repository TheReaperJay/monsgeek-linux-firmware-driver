    **1. Create `src/flow_control.rs`** implementing echo-matched query and fire-and-forget send:

    ```rust
    use crate::error::TransportError;
    use crate::usb::UsbSession;
    use monsgeek_protocol::{build_command, ChecksumType};
    use monsgeek_protocol::timing;
    use std::time::Duration;
    ```

    Implement `query_command(session: &UsbSession, cmd_byte: u8, data: &[u8], checksum: ChecksumType) -> Result<[u8; 64], TransportError>`:
    - Build 65-byte frame via `build_command(cmd_byte, data, checksum)`
    - Loop up to `timing::QUERY_RETRIES` (5) times:
      - Call `session.vendor_set_report(&frame[1..])` (skip report ID byte 0, send 64 bytes)
      - Sleep `Duration::from_millis(timing::DEFAULT_DELAY_MS)` (100ms) for firmware to process
      - Call `session.vendor_get_report()` to read 64-byte response
      - If `response[0] == cmd_byte`, return `Ok(response)` (echo matched)
      - Otherwise log debug: "Echo mismatch attempt {n}: expected 0x{cmd_byte:02X}, got 0x{response[0]:02X}"
    - After all retries exhausted, return `Err(TransportError::EchoMismatch { expected: cmd_byte, actual: last_response[0], attempts: timing::QUERY_RETRIES })`

    Implement `send_command(session: &UsbSession, cmd_byte: u8, data: &[u8], checksum: ChecksumType) -> Result<(), TransportError>`:
    - Build 65-byte frame via `build_command(cmd_byte, data, checksum)`
    - Loop up to `timing::SEND_RETRIES` (3) times:
      - Call `session.vendor_set_report(&frame[1..])` (skip report ID)
      - On Ok, return Ok(())
      - On Err, log warning and retry
    - After all retries, return the last error

    CRITICAL: The `build_command` return is 65 bytes `[report_id=0, payload...]`. rusb control transfers encode report ID in wValue, so we send `&frame[1..]` (64 bytes). This is documented in the 02-RESEARCH.md Pitfall 2.

    **1. Create `src/thread.rs`** with the dedicated transport thread:

    Define types for the command channel:
    ```rust
    use crossbeam_channel::{bounded, Receiver, Sender, select};
    use std::time::{Duration, Instant};

    /// Request sent from TransportHandle to the transport thread.
    pub(crate) struct CommandRequest {
        pub cmd: u8,
        pub data: Vec<u8>,
        pub checksum: monsgeek_protocol::ChecksumType,
        pub is_query: bool,
        pub response_tx: Sender<Result<Option<[u8; 64]>, TransportError>>,
    }

    /// Events emitted by the transport thread for device lifecycle.
    #[derive(Debug, Clone)]
    pub enum TransportEvent {
        DeviceArrived {
            vid: u16,
            pid: u16,
            bus: u8,
            address: u8,
        },
        DeviceLeft {
            bus: u8,
            address: u8,
        },
    }
    ```

    Implement `spawn_transport_thread`:

    ```rust
    pub(crate) fn spawn_transport_thread(
        cmd_rx: Receiver<CommandRequest>,
        event_tx: Sender<TransportEvent>,
        session: UsbSession,
    ) -> std::thread::JoinHandle<()>
    ```

    The thread function:
    - Track `last_command: Instant` initialized to `Instant::now() - Duration::from_millis(200)` (allows first command immediately)
    - Loop: `while let Ok(req) = cmd_rx.recv()`:
      - Enforce minimum inter-command delay: `let elapsed = last_command.elapsed(); let min_delay = Duration::from_millis(timing::DEFAULT_DELAY_MS); if elapsed < min_delay { std::thread::sleep(min_delay - elapsed); }`
      - If `req.is_query`: call `flow_control::query_command(&session, req.cmd, &req.data, req.checksum)`, send `Ok(Some(response))` or error via `req.response_tx`
      - If not query: call `flow_control::send_command(&session, req.cmd, &req.data, req.checksum)`, send `Ok(None)` or error via `req.response_tx`
      - Update `last_command = Instant::now()`
    - When channel closes (recv returns Err), log info "Transport thread shutting down" and return

    Implement hot-plug registration (if `rusb::has_hotplug()` is true):

    ```rust
    struct HotplugWatcher {
        event_tx: Sender<TransportEvent>,
    }

    impl<T: rusb::UsbContext> rusb::Hotplug<T> for HotplugWatcher {
        fn device_arrived(&mut self, device: rusb::Device<T>) {
            if let Ok(desc) = device.device_descriptor() {
                if desc.vendor_id() == 0x3141 {
                    let _ = self.event_tx.send(TransportEvent::DeviceArrived {
                        vid: desc.vendor_id(),
                        pid: desc.product_id(),
                        bus: device.bus_number(),
                        address: device.address(),
                    });
                }
            }
        }

        fn device_left(&mut self, device: rusb::Device<T>) {
            let _ = self.event_tx.send(TransportEvent::DeviceLeft {
                bus: device.bus_number(),
                address: device.address(),
            });
        }
    }
    ```

    CRITICAL per RESEARCH Pitfall 5: Do NOT call any DeviceHandle methods (control transfers, descriptor reads beyond device_descriptor) inside hotplug callbacks. Only send notification events via channel. Device opening happens in the main transport thread after receiving the event.

    Implement `spawn_hotplug_thread(event_tx: Sender<TransportEvent>) -> Option<std::thread::JoinHandle<()>>`:
    - Check `rusb::has_hotplug()`. If false, log warn "Hot-plug not supported on this platform" and return None.
    - Create `rusb::Context::new()`
    - Register `HotplugBuilder::new().vendor_id(0x3141).enumerate(true).register(&context, Box::new(HotplugWatcher { event_tx }))`
    - Loop: `context.handle_events(Some(Duration::from_millis(500)))` until error or thread is stopped
    - Return `Some(join_handle)`

    **2. Update `src/lib.rs`** to add TransportHandle as the public API:


    Define the `TransportHandle` struct:
    ```rust
    use crossbeam_channel::{bounded, Sender, Receiver};
    use monsgeek_protocol::ChecksumType;

    /// Handle for sending commands to a connected keyboard.
    ///
    /// Clone this handle to share across async tasks or threads.
    /// All commands are serialized through the transport thread.
    #[derive(Clone)]
    pub struct TransportHandle {
        cmd_tx: Sender<thread::CommandRequest>,
    }
    ```

    Implement methods on TransportHandle:
    - `pub fn send_query(&self, cmd: u8, data: &[u8], checksum: ChecksumType) -> Result<[u8; 64], TransportError>`: Create a bounded(1) response channel, send `CommandRequest { cmd, data: data.to_vec(), checksum, is_query: true, response_tx }` through `cmd_tx`, recv from response_rx, unwrap the inner Option (query always returns Some). Map channel recv error to `TransportError::ChannelClosed`.
    - `pub fn send_fire_and_forget(&self, cmd: u8, data: &[u8], checksum: ChecksumType) -> Result<(), TransportError>`: Same as above but `is_query: false`, ignore the None response body.
    - `pub fn shutdown(self)`: Drop the cmd_tx sender (closing the channel causes the transport thread to exit its recv loop).

    Implement a builder/connect function:
    ```rust
    /// Connect to a keyboard and start the transport thread.
    ///
    /// Returns (TransportHandle, Receiver<TransportEvent>).
    /// The event receiver emits hot-plug and device lifecycle events.
    pub fn connect(vid: u16, pid: u16) -> Result<(TransportHandle, Receiver<TransportEvent>), TransportError> {
        let session = UsbSession::open(vid, pid)?;
        let (cmd_tx, cmd_rx) = bounded(monsgeek_protocol::timing::dongle::REQUEST_QUEUE_SIZE);
        let (event_tx, event_rx) = bounded(32);

        thread::spawn_transport_thread(cmd_rx, event_tx.clone(), session);
        thread::spawn_hotplug_thread(event_tx);

        Ok((TransportHandle { cmd_tx }, event_rx))
    }
    ```
