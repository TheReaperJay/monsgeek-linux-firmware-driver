Implement `resolve_target_device` with this exact precedence:
1. `--path`
2. `--device-id`
3. `--model`
4. auto-single (only when one supported online device)

If more than one supported online device exists and no selector is provided, return non-zero exit with a message that includes all three selector forms (`--path`, `--device-id`, `--model`).

Load registry from:
- `MONSGEEK_DEVICE_REGISTRY_DIR` when set
- otherwise `${CARGO_MANIFEST_DIR}/../monsgeek-protocol/devices`

Map typed commands to raw message payloads with concrete command bytes/checksums:
- `info` -> send `[GET_USB_VERSION]` with Bit7
- `led get` -> send `[GET_LEDPARAM]` with Bit7
- `led set` -> send `[SET_LEDPARAM, mode, (4-speed), brightness, option, r, g, b]` with Bit8
  - `option=7` when dazzle is enabled
  - `option=8` when dazzle is disabled
- `debounce get/set` -> use `definition.commands().get_debounce` / `set_debounce`
  - set payload must follow family behavior:
    - YiChip byte `0x11`: payload `[0, value]`
    - otherwise payload `[value]`
  - checksum Bit7
- `poll get/set` -> use `definition.commands().get_report`/`set_report` when present, else fallback to shared `GET_REPORT`/`SET_REPORT`; checksum Bit7
- `profile get/set` -> use `definition.commands().get_profile`/`set_profile`; checksum Bit7
- `keymap get` -> send `[commands.get_keymatrix, profile, key_index, 0, 0]` with Bit7
- `keymap set` -> send `[commands.set_keymatrix, profile, key_index, 0, 0, 1, layer, 0, config_type, b1, b2, b3]` with Bit7
- `macro get` -> send `[GET_MACRO, macro_index, page]` with Bit7
- `macro set` -> send `[commands.set_macro, macro_index, page, chunk_len, is_last, 0, 0, data...]` with Bit7
- `raw send` -> requires `--unsafe` when first byte is a write command (`< 0x80`); `raw read` does not require `--unsafe`

Add output helpers in `format.rs` so every command supports both human-readable and JSON output (`--json`).
