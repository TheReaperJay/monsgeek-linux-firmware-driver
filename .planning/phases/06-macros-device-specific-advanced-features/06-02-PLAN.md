SET_MACRO wire format (from reference):
- Header: [set_macro_cmd, macro_index, chunk_page, total_pages, len_lo, len_hi, data_len, checksum]
- Payload: variable-length macro event data after header
- Checksum: Bit7 over full 64-byte frame

GET_MACRO wire format:
- Query: [GET_MACRO, macro_index, page]
- Response: 64-byte frame, response[0] = GET_MACRO echo

Magnetic wire format (from reference):
- GET_MULTI_MAGNETISM query: [GET_MULTI_MAGNETISM, subcmd, page]
- SET_MULTI_MAGNETISM: [SET_MULTI_MAGNETISM, subcmd, page, pad*4, checksum, key_data...]
  where key_data is 2-byte LE per key
- SET_KEY_MAGNETISM_MODE: [SET_KEY_MAGNETISM_MODE, key_index, mode, 0, 0, 0, 0, checksum]
- SET_MAGNETISM_CAL: [SET_MAGNETISM_CAL, start_or_stop(1/0), 0*5, checksum]

Follow the established read-write-readback-restore pattern:

1. `require_dangerous_write_opt_in()` + `HW_LOCK`
2. Connect to M5W, get `commands = m5w.commands()`
3. Use slot 0 (avoids GET_MACRO stride bug on some firmware — see research)
4. Read original: `handle.send_query(cmd::GET_MACRO, &[0, 0], ChecksumType::Bit7)` — save full response as `original_page0`
5. Build a test macro payload: a simple 1-page macro with recognizable test pattern.
   Construct a 64-byte SET_MACRO frame:
   - `msg[0] = commands.set_macro` (device-specific command byte)
   - `msg[1] = 0` (macro_index = slot 0)
   - `msg[2] = 0` (chunk_page = 0)
   - `msg[3] = 1` (total_pages = 1)
   - `msg[4] = 8` (len_lo: 8 bytes of macro data)
   - `msg[5] = 0` (len_hi)
   - `msg[6] = 8` (data_len for this chunk)
   - Fill msg[7..15] with recognizable test pattern: `[0xDE, 0xAD, 0xBE, 0xEF, 0x01, 0x02, 0x03, 0x04]`
   Send with `handle.send_fire_and_forget(commands.set_macro, &msg[1..], ChecksumType::Bit7)` (payload excludes command byte since send_fire_and_forget prepends it)
6. Read back: `handle.send_query(cmd::GET_MACRO, &[0, 0], ChecksumType::Bit7)` — compare relevant data bytes against test pattern
7. Restore: reconstruct SET_MACRO from `original_page0` data and send. The restore payload should mirror the original macro data.
8. Verify restore: read again and compare against original

Use the closure pattern from debounce test: `test_result` closure for steps 5-6, `restore_result` closure for step 7-8. Always attempt restore even if test fails. Print diagnostic info at each step.

**Important notes for executor:**
- `send_fire_and_forget` takes `(cmd_byte, payload_without_cmd, checksum_type)` — the transport layer prepends the command byte and builds the frame
- `send_query` takes `(cmd_byte, query_params, checksum_type)` — query params are the bytes after the command byte
- The transport layer applies 100ms throttling between commands automatically
- Use `ChecksumType::Bit7` for all macro commands (per reference implementation)
- The command byte for SET_MACRO must come from `m5w.commands().set_macro` (not `cmd::SET_MACRO` directly) to handle YiChip overrides
