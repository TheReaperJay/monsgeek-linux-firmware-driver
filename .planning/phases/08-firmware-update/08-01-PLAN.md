Implement `preflight.rs` with concrete policy:
- `PreflightRequest` includes:
  - selected device identity (`device_id`, `model_slug`, optional path)
  - firmware input source (`local_file` or `vendor_download`)
  - `allow_unofficial: bool`
  - `assume_yes: bool`
  - `high_risk_ack: bool`
  - `typed_phrase: Option<String>`
  - optional backup outcome (`backup_attempted`, `backup_ok`)
- `PreflightDecision` includes `allowed: bool`, `errors: Vec<String>`, `warnings: Vec<String>`, and normalized manifest summary (size/chunks/checksum).

Enforce exact gates:
1. Reject empty or unreadable file.
2. Reject missing/invalid manifest compatibility fields.
3. Reject metadata mismatch unless `allow_unofficial == true`.
4. If interactive path (`assume_yes == false`), require exact typed phrase: `FLASH M5W`.
5. If non-interactive (`assume_yes == true`), require `high_risk_ack == true`; `--yes` alone must fail.
6. If backup was attempted and failed, add blocking error unless a second explicit override marker is provided (`allow_backup_failure == true`).

Implement checksum helper in shared crate that sums firmware bytes plus 0xFF padding up to 64-byte boundary, then emits `u32` with lower 24-bit comparison helper.
