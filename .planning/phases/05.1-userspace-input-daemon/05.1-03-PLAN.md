    Opens a `UsbSession` with `SessionMode::InputOnly` for the M5W. Verifies:
    - Session opens without error
    - `read_report_with_timeout` with 100ms timeout returns Ok (either data or 0 bytes for timeout)

    **Test 2: `test_uinput_device_creation`**
    Creates a uinput virtual device via `create_uinput_device("test-monsgeek-inputd")`. Verifies:
    - Device creation succeeds
    - Device can be dropped without error

    **Test 3: `test_daemon_coexistence`**
    Verifies daemon and bridge can both hold their interfaces simultaneously:
    1. Open a `UsbSession` with `SessionMode::InputOnly` (claims IF0/IF1)
    2. Open a second `UsbSession` with `SessionMode::ControlOnly` (claims IF2)
    3. Read from IF0 on the first session (should return Ok with timeout)
    4. Drop both sessions
    This proves the coexistence model works at the USB level.

    **Test 4: `test_input_poll_produces_actions`**
    Opens InputOnly session, creates `InputProcessor`, polls for 2 seconds in a loop.
    Prints any key actions received (informational — user must press a key during test).
    Test always passes (it's a diagnostic); actual functionality verified in checkpoint.

    Each test logs via `simple_logger` at debug level for diagnostics.
