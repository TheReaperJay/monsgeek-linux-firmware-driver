  <action>
    Create `crates/monsgeek-inputd/src/daemon.rs` with the full daemon lifecycle.

    The daemon does NOT use the transport thread / `TransportHandle` / `CommandController` machinery. It directly uses `UsbSession` for IF0 reads and `InputProcessor` for report processing. This avoids the command channel, 100ms throttling, and IF2 vendor logic that are irrelevant to input processing.

    **Public API:**
    ```rust
    pub struct DaemonConfig {
        pub debounce_ms: u64,
        pub device_filter: Option<(u8, u8)>,  // Optional bus:addr filter
        pub device_name: String,              // uinput device name
    }

    pub fn run_daemon(config: DaemonConfig) -> Result<(), Box<dyn std::error::Error>>;
    ```

    **Signal handling setup (at top of `run_daemon`):**
    ```rust
    use signal_hook::consts::{SIGINT, SIGTERM};
    use signal_hook::flag;
    use std::sync::Arc;
    use std::sync::atomic::{AtomicBool, Ordering};

    let shutdown = Arc::new(AtomicBool::new(false));
    flag::register(SIGTERM, Arc::clone(&shutdown))?;
    flag::register(SIGINT, Arc::clone(&shutdown))?;
    ```

    **Device discovery:**
    Load the device registry from `crates/monsgeek-protocol/devices/` (use `DeviceRegistry::load_from_directory`). Find the first connected MonsGeek keyboard using `discovery::enumerate_devices`. If `device_filter` is set, find by matching bus:address. Use the found device's VID/PID for `UsbSession::open_with_mode`.

    Note: The daemon currently targets the M5W (device ID 1308) as the only verified keyboard. If multiple devices are found and no `--device` filter is provided, use the first one and log a warning suggesting `--device`.

    **Main outer loop (connect/reconnect):**
    ```
    while !shutdown.load(Ordering::Relaxed) {
        match try_connect_and_run(&config, &shutdown) {
            Ok(()) => break,  // Clean shutdown
            Err(DaemonError::Disconnected) => {
                // Keyboard unplugged — wait for reconnect
                sd_notify STATUS="Waiting for keyboard"
                wait_for_device_udev(&shutdown, vid)?;
                // Loop back to try_connect_and_run
            }
            Err(e) => return Err(e.into()),
        }
    }
    ```

    **`try_connect_and_run` function:**
    1. Load registry, enumerate devices, find target keyboard
    2. Open `UsbSession::open_with_mode(vid, pid, SessionMode::InputOnly)`
    3. Create fresh `InputProcessor::new(config.debounce_ms)` — CRITICAL: fresh instance per connect to clear stale debounce state (see RESEARCH.md Pitfall 5)
    4. Create uinput device via `create_uinput_device(&config.device_name)`
    5. Send `sd_notify::notify(false, &[NotifyState::Ready])` and `NotifyState::Status("Connected to {display_name} ({device_id})")`
    6. Enter poll loop

    **Poll loop (inner hot path):**
    ```rust
    let mut report = [0u8; 8];
    while !shutdown.load(Ordering::Relaxed) {
        match session.read_report_with_timeout(&mut report, Duration::from_millis(10)) {
            Ok(n) if n >= 8 => {
                let actions = processor.process_report(&report);
                if !actions.is_empty() {
                    if let Err(e) = emit_actions(&mut uinput_dev, &actions) {
                        log::error!("uinput emit failed: {}", e);
                        // uinput failure is fatal for input path
                        break;
                    }
                }
            }
            Ok(_) => {} // Timeout or short read, continue polling
            Err(TransportError::Disconnected) => {
                log::warn!("Keyboard disconnected");
                let release_actions = processor.release_all_keys();
                let _ = emit_actions(&mut uinput_dev, &release_actions);
                drop(uinput_dev);  // Destroy uinput device
                drop(session);     // Release IF0/IF1, reattach kernel
                return Err(DaemonError::Disconnected);
            }
            Err(e) => {
                log::debug!("IF0 poll error: {}", e);
            }
        }
    }
    // Clean shutdown path
    let release_actions = processor.release_all_keys();
    let _ = emit_actions(&mut uinput_dev, &release_actions);
    // uinput_dev and session are dropped here, releasing resources
    Ok(())
    ```

    **`wait_for_device_udev` function:**
    Follow the existing pattern from `thread.rs::spawn_hotplug_thread`:
    1. Create `udev::MonitorBuilder::new()?.match_subsystem("usb")?.listen()?`
    2. Use `libc::poll` on the monitor fd with 1-second timeout
    3. Check `shutdown` flag each iteration
    4. When a device add event matches the target VID, return

    **Error type:**
    ```rust
    enum DaemonError {
        Disconnected,
        Transport(TransportError),
        Io(std::io::Error),
        Registry(String),
    }
    ```
    Implement `From<TransportError>`, `From<std::io::Error>`, and `Display` for `DaemonError`.

    **sd_notify watchdog:**
    In the poll loop, send `sd_notify::notify(false, &[NotifyState::Watchdog])` periodically (e.g., every 5 seconds using an `Instant` tracker). This satisfies systemd's `WatchdogSec=` if configured.

    ```rust
    fn main() {
        env_logger::init();
        let cli = Cli::parse();
        let config = config::load_config();
        let debounce_ms = config::resolve_debounce_ms(cli.debounce_ms, &config);

        let device_filter = cli.device.as_ref().and_then(|d| {
            let parts: Vec<&str> = d.split(':').collect();
            if parts.len() == 2 {
                match (parts[0].parse::<u8>(), parts[1].parse::<u8>()) {
                    (Ok(bus), Ok(addr)) => Some((bus, addr)),
                    _ => {
                        log::error!("Invalid --device format '{}', expected BUS:ADDR (e.g., 001:005)", d);
                        std::process::exit(1);
                    }
                }
            } else {
                log::error!("Invalid --device format '{}', expected BUS:ADDR (e.g., 001:005)", d);
                std::process::exit(1);
            }
        });

        let daemon_config = daemon::DaemonConfig {
            debounce_ms,
            device_filter,
            device_name: "MonsGeek M5W (monsgeek-inputd)".to_string(),
        };

        if let Err(e) = daemon::run_daemon(daemon_config) {
            log::error!("Daemon exited with error: {}", e);
            std::process::exit(1);
        }
    }
    ```
