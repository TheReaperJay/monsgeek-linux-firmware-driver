    **1. Create `crates/monsgeek-protocol/src/registry.rs`:**

    ```rust
    use std::collections::HashMap;
    use std::path::Path;
    use crate::device::DeviceDefinition;
    use crate::error::RegistryError;
    ```

    Implement `DeviceRegistry` struct:
    ```rust
    pub struct DeviceRegistry {
        devices_by_id: HashMap<i32, DeviceDefinition>,
        devices_by_vid_pid: HashMap<(u16, u16), Vec<i32>>,
    }
    ```

    Methods:
    - `pub fn new() -> Self` -- empty registry
    - `pub fn load_from_directory(dir: &Path) -> Result<Self, RegistryError>` -- uses `glob::glob(dir.join("*.json").to_str()...)` to find all JSON files, reads each with `std::fs::read_to_string`, deserializes with `serde_json::from_str::<DeviceDefinition>`, calls `self.add_device()`. Returns error if glob pattern fails or if a JSON file fails to parse. Empty directory returns Ok with 0 devices.
    - `fn add_device(&mut self, device: DeviceDefinition)` -- inserts into `devices_by_id` by `device.id`, pushes `device.id` into `devices_by_vid_pid` entry for `(device.vid, device.pid)`
    - `pub fn find_by_id(&self, id: i32) -> Option<&DeviceDefinition>` -- HashMap lookup
    - `pub fn find_by_vid_pid(&self, vid: u16, pid: u16) -> Vec<&DeviceDefinition>` -- look up `(vid, pid)` in vid_pid map, resolve each i32 ID to DeviceDefinition reference, return vec. Return empty vec if key not found.
    - `pub fn len(&self) -> usize` -- count of devices
    - `pub fn is_empty(&self) -> bool`
    - `pub fn all_devices(&self) -> impl Iterator<Item = &DeviceDefinition>` -- iterate all
